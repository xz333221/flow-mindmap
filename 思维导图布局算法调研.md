## 3. 推荐算法：Reingold–Tilford 树布局（含左右分区）
### 3.1 算法目标
Reingold–Tilford “tidy tree” 算法要满足：
1. 同一层的节点在同一水平线上（深度决定 y）。
2. 父节点在所有子节点的水平中间位置。
3. 子树之间不重叠，且尽量紧凑。
4. 同构子树画出来是对称的。
D3 的 `d3.hierarchy.tree()` 就是这个算法的线性时间实现。
### 3.2 简化版思路（先理解逻辑）
1. **后序遍历**（先处理所有子树，再处理当前节点）：
   - 递归计算每个子树的宽度。
   - 为每个子节点分配一个“临时 x”（例如按兄弟顺序 0,1,2…）。
   - 当前节点的 `prelim`（初步 x）设为子节点 x 的中间值。
2. **处理子树重叠**：
   - 比较相邻子树的“轮廓”（左轮廓/右轮廓），如果重叠就把右子树往右推。
3. **前序遍历**修正：
   - 从根开始，累加祖先的 `modifier`，得到每个节点的最终 `x`。
### 3.3 加上“左右分支”思维导图风格
经典 Reingold–Tilford 是整棵树往右展开；思维导图通常是：
- 根节点居中。
- 一级子节点一部分向左，一部分向右。
- 二级及以下跟随父节点方向。
**做法：**
1. 给每个节点增加一个方向属性：`dir: 'left' | 'right' | 'root'`。
2. 递归时，按 `dir` 把子节点分成 **左子树** 和 **右子树**。
3. 对左子树做一次镜像 Reingold–Tilford（x 取反），对右子树做一次正常 Reingold–Tilford。
4. 根节点 `x=0`，左右子树分别向左/右展开。
伪代码（简化版）：
```ts
type Dir = 'left' | 'right' | 'root';
interface Node {
  id: string;
  children: Node[];
  dir: Dir;
  // 布局临时变量
  x: number;
  y: number;
  prelim: number;
  mod: number;
}
const SIBLING_GAP = 20;   // 兄弟间距
const LEVEL_GAP = 80;     // 层级间距（y 方向）
function layoutMindMap(root: Node) {
  // 1. 先按深度设置 y
  function setY(node: Node, depth: number) {
    node.y = depth * LEVEL_GAP;
    node.children.forEach(c => setY(c, depth + 1));
  }
  setY(root, 0);
  // 2. 递归计算每个子树的布局（带方向）
  function layoutSubtree(node: Node) {
    if (!node.children.length) {
      node.prelim = 0;
      return;
    }
    // 按 dir 分左右子节点
    const leftChildren = node.children.filter(c => c.dir === 'left');
    const rightChildren = node.children.filter(c => c.dir === 'right');
    // 先递归布局子树
    leftChildren.forEach(c => layoutSubtree(c));
    rightChildren.forEach(c => layoutSubtree(c));
    // 对右子树：正常 Reingold–Tilford
    layoutHorizontalList(rightChildren);  // 见下面
    // 对左子树：镜像（x 取反）
    layoutHorizontalList(leftChildren);
    leftChildren.forEach(c => mirrorSubtree(c)); // x = -x
    // 把左右子树拼到当前节点两侧
    const rightWidth = rightChildren.length
      ? rightChildren[rightChildren.length - 1].prelim + nodeWidth(rightChildren[rightChildren.length - 1])
      : 0;
    const leftWidth = leftChildren.length
      ? -leftChildren[0].prelim  // 因为镜像后左子树最右端在 0
      : 0;
    // 当前节点居中于左右子树之间
    node.prelim = (leftWidth + rightWidth) / 2;
  }
  // 简化：对一组同方向子节点做水平排列（不考虑子树间轮廓冲突，先有个能跑的版本）
  function layoutHorizontalList(children: Node[]) {
    let nextX = 0;
    children.forEach(child => {
      child.prelim = nextX;
      nextX += nodeWidth(child) + SIBLING_GAP;
    });
  }
  // 镜像子树：x 全部取反
  function mirrorSubtree(node: Node) {
    node.prelim = -node.prelim;
    node.children.forEach(c => mirrorSubtree(c));
  }
  function nodeWidth(node: Node) {
    // 可以根据文本长度动态计算，这里先写死
    return 120;
  }
  // 3. 第一次后序遍历：计算 prelim
  layoutSubtree(root);
  // 4. 第二次前序遍历：累加 mod，得到最终 x
  function accumulateMod(node: Node, modSum: number) {
    node.x = node.prelim + modSum;
    node.children.forEach(c => accumulateMod(c, modSum + (c.mod || 0)));
  }
  accumulateMod(root, 0);
  // 5. 根节点居中到画布中央（可选）
  // root.x = canvasWidth / 2;
}
```
这个版本没有做“子树轮廓冲突检测”，但对于不太宽的导图已经够用。

## 4. 更通用的层次布局：Sugiyama 算法（适合流程图、组织架构）
如果你还要支持 **流程图、组织架构图、有向无环图**，而不只是纯树，可以用 **Sugiyama 分层布局**：
核心 4 步：
1. **去环**：如果有环，先反转部分边，变成 DAG。
2. **分层（layering）**：给每个节点分配层级（y 坐标）。
3. **交叉最小化（crossing minimization）**：在每一层内调整节点顺序，减少边交叉。
4. **坐标分配（x 坐标）**：在每层内分配 x，使边尽量短、尽量垂直。
JS 里可以直接用：
- `dagre`：基于 Sugiyama 的 JS 布局库。
- `elkjs`：更强大的分层布局引擎。
Sugiyama 比 Reingold–Tilford 更重，但适合 **多种布局类型混用** 的思维导图工具（比如 XMind 支持逻辑图、组织架构图、鱼骨图等）。

## 5. 径向/放射状思维导图布局
如果你想要 **“根在圆心，分支按角度展开”** 的效果：
思路：
1. 仍然用树结构，**y 对应半径**，**x 对应角度**。
2. 对根节点：`r=0`，`angle=0`。
3. 递归时，把子节点按角度均匀分布在一个圆环上：
```ts
function layoutRadial(node: Node, radius: number, startAngle: number, endAngle: number) {
  node.x = radius * Math.cos((startAngle + endAngle) / 2);  // 或直接用角度
  node.y = radius * Math.sin((startAngle + endAngle) / 2);
  if (!node.children.length) return;
  const angleStep = (endAngle - startAngle) / node.children.length;
  node.children.forEach((child, i) => {
    const childStart = startAngle + i * angleStep;
    const childEnd = childStart + angleStep;
    layoutRadial(child, radius + RADIUS_STEP, childStart, childEnd);
  });
}
```
D3、ECharts、ArcGIS 的径向树布局都是类似思路。

## 1. 经典树状布局算法 (Tree Layout)

思维导图在数据结构上本质是一个**具名树（Tree）**（一个根节点，多个子节点，无环）。因此，最基础也最常用的算法是**树状布局算法**。

其中最著名的经典算法是 **Reingold-Tilford 算法**。

### 实现思路（以标准的左右分布思维导图为例）：

这个算法的核心思想是“自底向上计算尺寸，自顶向下分配坐标”。通常分为两个递归步骤：

1. **第一次遍历（自底向上 - Post-order Traversal）：**
* 从叶子节点开始，计算每个节点自身的高度和宽度。
* 父节点的高度取决于它所有子节点高度的总和（加上节点之间的间距 `Sibling Distance`）。
* 通过这种方式，计算出以当前节点为根的“子树”所占用的总垂直空间。


2. **第二次遍历（自顶向下 - Pre-order Traversal）：**
* 根据父节点的位置，将子节点在垂直方向上均匀平分（居中对齐）。
* 确定 X 轴坐标：子节点的 $X = \text{父节点的 } X + \text{父节点宽度} + \text{层间距 (Level Distance)}$。



> **优点：** 性能极高（时间复杂度 $O(n)$），结构非常清晰、紧凑，绝对不会出现节点重叠。
> **缺点：** 属于规则几何布局，如果某一个分支子节点特别多，会让画布在某一个方向上拉得很长。

## 2. 力导向布局算法 (Force-Directed Layout)

如果你希望你的思维导图有一种**弹性、自然流淌、支持任意拖拽回弹**的炫酷效果（类似 Obsidian 的关系图谱，或者某些脑图的自由布局模式），你可以使用**力导向算法**（如 Fruchterman-Reingold 算法）。

### 实现思路：

把整个思维导图看作一个物理系统：

* **库仑斥力：** 所有节点之间都存在排斥力，防止它们挨得太近。
* **胡克弹力：** 有连线（父子关系）的节点之间存在引力，像弹簧一样把它们拉在一起。

通过在每一帧循环（Canvas 或 WebGL 动画）中计算每个节点受到的合力，更新它们的速度和位置，直到整个系统达到能量平衡。

> **优点：** 视觉效果非常动感、现代；适应性强，不需要预设规则。
> **缺点：** 计算量大（通常是 $O(n^2)$ 或 $O(n \log n)$），节点多时会卡顿；位置不固定，每次打开可能长得不太一样，不适合传统的规范脑图。

```javascript
// 简化版的逻辑示例
function layout(node) {
    // 1. 如果是叶子节点，它占用的总高度就是它自身的高度
    if (!node.children || node.children.length === 0) {
        node.subTreeHeight = node.height;
        return;
    }

    // 2. 递归计算所有子树的高度
    let totalChildrenHeight = 0;
    node.children.forEach(child => {
        layout(child);
        totalChildrenHeight += child.subTreeHeight;
    });
    
    // 加上子节点之间的间距 (gap)
    totalChildrenHeight += (node.children.length - 1) * 20; 

    // 3. 当前节点的子树总高度，取决于自身和子节点总高度的最大值
    node.subTreeHeight = Math.max(node.height, totalChildrenHeight);
}

function assignCoordinates(node, currentX, currentY) {
    node.x = currentX;
    node.y = currentY;

    if (!node.children) return;

    // 计算子节点开始放置的起始 Y 坐标（为了让子节点整体相对于父节点居中）
    let startY = currentY + node.height / 2 - node.subTreeHeight / 2;
    
    node.children.forEach(child => {
        let childX = currentX + node.width + 50; // 50 是层间距
        let childY = startY + child.subTreeHeight / 2 - child.height / 2;
        
        assignCoordinates(child, childX, childY);
        
        // 下一个子节点的起始 Y 坐标累加
        startY += child.subTreeHeight + 20; // 20 是兄弟节点间距
    });
}

```

### 1. 基础算法：树形布局 (Tree Layout)
这是最经典、最基础的算法，适用于**逻辑图**（向右/左展开）和**组织结构图**（向下展开）。

*   **核心思想**：自底向上计算子树尺寸，自顶向下分配坐标。
*   **实现步骤**：
    1.  **后序遍历 (Post-order)**：从叶子节点开始，计算每个节点及其子树的**宽度**（或高度）。叶子节点的尺寸就是自身尺寸；非叶子节点的尺寸 = 自身尺寸 + 所有子节点尺寸 + 子节点间的间距。
    2.  **前序遍历 (Pre-order)**：从根节点开始，根据父节点的位置和已计算好的子树尺寸，依次累加间距，确定每个子节点的坐标。
*   **优点**：实现简单，时间复杂度 $O(N)$。
*   **缺点**：如果某个分支特别庞大，会导致该分支下的节点间距被拉得很大，或者与其他分支重叠（如果处理不好）。

### 2. 进阶算法：紧凑树布局 (Compact Tree / Reingold-Tilford)
为了解决基础树形布局的“空间浪费”和“重叠”问题，Reingold-Tilford 算法是学术界和工业界的标杆。

*   **核心思想**：在分配坐标时，不仅考虑当前子树，还要考虑**兄弟子树之间的轮廓**，尽量让子树靠拢，同时保证不重叠。
*   **关键点**：
    *   维护每个子树的**左轮廓**和**右轮廓**。
    *   在合并兄弟子树时，通过轮廓计算最小安全距离。
    *   引入**Apex（顶点）**或**Thread（线程）**概念，处理深层嵌套时的居中问题。
*   **开源参考**：D3.js 的 `d3-hierarchy` 中的 `tree()` 和 `cluster()` 布局，以及 `d3-flextree`。

### 3. 思维导图专属：脑图布局 (Mindmap / Radial Layout)
这是 XMind、MindNode 等软件最常用的布局，根节点在中间，向四周发散。

*   **核心思想**：将节点分配到不同的**象限**或**扇区**，根据子树权重分配角度。
*   **实现步骤**：
    1.  **划分区域**：通常将 360 度按子节点数量或子树权重（叶子节点数）比例分配。
    2.  **极坐标转换**：
        *   角度 $\theta$ = 当前扇区的中心角。
        *   半径 $r$ = 基础半径 + 层级深度 $\times$ 层级间距。
        *   $x = r \cdot \cos(\theta)$, $y = r \cdot \sin(\theta)$。
    3.  **防重叠处理**：如果两个分支角度太近，需要动态调整角度或增加半径，甚至将冲突的分支“踢”到另一个半圆。
*   **难点**：处理奇数个子节点时的对称性，以及长文本节点的包围盒碰撞检测。

### 4. 动态调整：力导向布局 (Force-Directed Layout)
通常不作为主布局，而是作为**辅助**或**自由模式**下的布局。

*   **核心思想**：模拟物理系统。节点间有斥力，连线有引力（弹簧）。
*   **适用场景**：当用户手动拖拽节点后，周围节点需要自动避让；或者在“大纲转导图”的过渡动画中。
*   **开源参考**：D3.js 的 `d3-force`。

    *   加上 Padding 和 Margin 后再参与布局计算。

3.  **第三步：引入“紧凑”逻辑**。
    *   在计算子节点 X 坐标时，不要简单累加 `prev.x + prev.width + gap`。
    *   尝试计算 `max(当前计算位置, 兄弟节点右边界 + gap)`，这是解决重叠的最简单手段。

4.  **第四步：连线绘制**。
    *   布局算出的是节点中心点或左上角。
    *   连线需要计算**锚点 (Anchor)**。例如：父节点从右侧出发，子节点从左侧进入。
    *   曲线推荐：**三次贝塞尔曲线** (Cubic Bezier)，控制点通常取水平方向的中点，这样曲线最平滑。

### ⚠️ 常见的坑

*   **浮点数精度**：布局计算全程用浮点数，最后渲染时再 `Math.round`，否则会有 1px 的抖动或缝隙。
*   **长文本**：如果某个节点文本极长，会导致该分支下的所有子节点都被挤到很远。解决方案：限制最大宽度，强制换行；或者在布局时给长文本节点一个“虚拟宽度”（比实际小），渲染时允许轻微重叠或自动缩放。
*   **动画过渡**：布局改变时，不要直接 `node.x = newX`，要用 `requestAnimationFrame` 做插值动画，否则用户体验极差。

## 二、核心算法：左右分布（最常用）

这是 XMind、SimpleMind 等软件的标准布局。核心思想是**后序遍历计算尺寸，前序遍历分配坐标**。

### 算法步骤

**第一步：后序遍历 —— 计算每个子树的"包围盒"**

```javascript
function computeLayout(node) {
    // 1. 先递归计算所有子节点
    node.children.forEach(child => computeLayout(child));
    
    // 2. 计算当前节点的布局尺寸
    let childrenWidth = 0;
    let childrenHeight = 0;
    
    if (node.children.length > 0) {
        // 兄弟节点间距
        const spacing = 20;
        // 所有子节点的内容宽度之和 + 间距
        childrenWidth = node.children.reduce((sum, c) => sum + c.layoutWidth, 0) 
                      + (node.children.length - 1) * spacing;
        // 子节点中最大的高度
        childrenHeight = Math.max(...node.children.map(c => c.layoutHeight));
    }
    
    // 3. 当前节点的布局尺寸 = max(自身尺寸, 子树尺寸)
    node.layoutWidth = Math.max(node.contentWidth, childrenWidth);
    node.layoutHeight = node.contentHeight + (node.children.length > 0 ? 40 + childrenHeight : 0);
    
    // 4. 计算当前节点在子树中的"中心偏移"（用于后续对齐）
    if (node.children.length >= 2) {
        const first = node.children[0];
        const last = node.children[node.children.length - 1];
        const firstCenter = first.cLeft;
        const lastCenter = node.layoutWidth - last.layoutWidth + last.cLeft;
        node.cLeft = (firstCenter + lastCenter) / 2; // 居中
    } else if (node.children.length === 1) {
        node.cLeft = Math.max(node.children[0].contentWidth, node.contentWidth) / 2;
    } else {
        node.cLeft = node.contentWidth / 2;
    }
}
```

**第二步：前序遍历 —— 分配实际坐标**

```javascript
function computePosition(node) {
    if (node.isRoot) {
        node.x = 0;
        node.y = 0;
    }
    
    if (node.children.length === 0) return;
    
    // 计算子节点起始位置（让子树整体居中于父节点）
    let startX = node.x + node.cLeft - node.layoutWidth / 2;
    const startY = node.y + node.contentHeight + 40; // 垂直间距
    
    node.children.forEach(child => {
        child.x = startX + child.cLeft - child.contentWidth / 2;
        child.y = startY;
        startX += child.layoutWidth + 20; // 水平间距
        
        // 递归设置子节点
        computePosition(child);
    });
}
```

**第三步：左右分配策略**

思维导图的关键是**根节点的子节点向左右两侧分布**：

```javascript
// 根节点的特殊处理：左右分配
function distributeRootChildren(root) {
    const leftChildren = [];
    const rightChildren = [];
    
    // 策略1：奇数节点，中间放右边；偶数节点，平分
    root.children.forEach((child, index) => {
        if (index % 2 === 0) {
            rightChildren.push(child);
        } else {
            leftChildren.push(child);
        }
    });
    
    // 分别计算左右两侧布局
    layoutSide(leftChildren, root, 'left');
    layoutSide(rightChildren, root, 'right');
}
```

**第四步：左右侧布局**

```javascript
function layoutSide(children, parent, direction) {
    // 计算每个子节点的 top 值（垂直居中分布）
    let totalHeight = children.reduce((sum, c) => sum + c.layoutHeight, 0);
    totalHeight += (children.length - 1) * 20; // 间距
    
    let currentTop = parent.y + parent.contentHeight / 2 - totalHeight / 2;
    
    children.forEach(child => {
        child.y = currentTop;
        // x 坐标：左侧在父节点左边，右侧在父节点右边
        if (direction === 'left') {
            child.x = parent.x - 100 - child.layoutWidth; // 水平间距
        } else {
            child.x = parent.x + parent.contentWidth + 100;
        }
        currentTop += child.layoutHeight + 20;
    });
}
```

## 三、放射状布局（Radial Layout）

根节点在中心，子节点按角度分布在同心圆上。这是 Eades 在 1991 年提出的经典算法。

### 核心公式

每个子树分配一个**扇形角度（wedge）**，角度大小与其子树叶子数成正比：

$$\tau_u = \frac{\ell(u)}{\ell(v) - 1} \times 2\pi$$

其中 $\ell(u)$ 是子树 $u$ 的叶子数，$\ell(v)$ 是父节点 $v$ 的所有子树叶子数之和。

### 伪代码

```javascript
function radialLayout(node, angleStart, angleEnd, radius, level) {
    // 计算节点坐标（极坐标转直角坐标）
    const angle = (angleStart + angleEnd) / 2;
    node.x = radius * Math.cos(angle);
    node.y = radius * Math.sin(angle);
    
    if (node.children.length === 0) return;
    
    // 计算总叶子数
    const totalLeaves = node.children.reduce((sum, c) => sum + countLeaves(c), 0);
    
    let currentAngle = angleStart;
    const radiusStep = 80; // 每层半径增量
    
    node.children.forEach(child => {
        const childLeaves = countLeaves(child);
        const wedgeAngle = (childLeaves / totalLeaves) * (angleEnd - angleStart);
        
        radialLayout(child, currentAngle, currentAngle + wedgeAngle, 
                    radius + radiusStep, level + 1);
        currentAngle += wedgeAngle;
    });
}
```

## 四、实用建议

1. **先算尺寸，再算位置**：这是所有树布局算法的通用模式。后序遍历计算每个子树的宽高，前序遍历分配坐标。

2. **处理节点折叠**：当节点折叠时，其子树尺寸为 0，布局会自动收缩。

3. **动画过渡**：布局重新计算后，用 CSS transition 或 requestAnimationFrame 平滑过渡，避免突兀跳动。

4. **碰撞检测**：如果节点内容很长，需要在布局阶段就考虑文本换行，避免节点重叠。

5. **参考开源实现**：
   - **simple-mind-map**：一个完整的思维导图库，实现了多种布局算法

## 二、辐射状布局（思维导图最常用）

根节点在中心，子节点按角度扇区分布在外围。关键是要 **按子树的“权重”分配角度，避免分支拥挤**。

### 2.1 算法步骤

1. **自底向上计算权重**  
   权重可定义为子树的叶子节点数量，或者子树占据的像素尺寸。叶子越多，分配的扇区角度越大。

2. **自顶向下分配角度区间**  
   根节点占据 360°（或自定义范围如 180°）。每个子节点根据权重瓜分父节点的角度区间。

3. **计算半径与坐标**  
   半径随层级递增。极坐标 `(radius, midAngle)` 转笛卡尔坐标。

### 2.2 关键伪代码

```javascript
// 1. 后序遍历计算子树权重（叶子节点数）
function calcWeight(node) {
  if (node.children.length === 0) {
    node.weight = 1;
    return 1;
  }
  let total = 0;
  node.children.forEach(child => {
    total += calcWeight(child);
  });
  node.weight = total;
  return total;
}

// 2. 自顶向下分配角度区间，并计算坐标
function radialLayout(node, radius, angleStart, angleEnd, layerSpacing) {
  // 设置当前节点的半径
  node.radius = radius;
  // 根节点放在中心
  if (angleStart === undefined) {
    node.x = 0;
    node.y = 0;
    // 根节点分配全角度（可限制为180或360）
    const childrenAngleRange = 2 * Math.PI; // 360°
    node.children.forEach(child => {
      radialLayout(child, radius + layerSpacing, 0, childrenAngleRange, layerSpacing);
    });
    return;
  }

  // 当前节点的极坐标角度取中值
  const midAngle = (angleStart + angleEnd) / 2;
  node.angle = midAngle;
  node.x = node.radius * Math.cos(midAngle);
  node.y = node.radius * Math.sin(midAngle);

  // 分配子节点角度区间
  const childCount = node.children.length;
  if (childCount === 0) return;

  // 总可用角度 = 父节点分配的角度范围
  const totalAngle = angleEnd - angleStart;
  
  // 按权重计算每个子节点的角度跨度
  let currentAngle = angleStart;
  for (let child of node.children) {
    const childRatio = child.weight / node.weight;  // 权重占比
    const childAngleSpan = totalAngle * childRatio;
    const childStart = currentAngle;
    const childEnd = currentAngle + childAngleSpan;
    
    radialLayout(child, radius + layerSpacing, childStart, childEnd, layerSpacing);
    currentAngle = childEnd;
  }
}

// 使用示例
const root = buildTree(); // 构建你的节点树
calcWeight(root);
const RADIUS_ROOT = 0;
const LAYER_SPACING = 120;  // 层间距，需大于节点尺寸
radialLayout(root, RADIUS_ROOT, undefined, undefined, LAYER_SPACING);
```

### 2.3 处理节点尺寸与重叠

上述算法仅按权重分配角度，但没有考虑节点的实际像素宽度。如果节点本身较宽，在弧线方向上可能重叠。进阶做法：

- 将权重改为 **子树所需的最小弧长**（即所有兄弟节点的“弧长需求”之和）。
- 弧长需求 ≈ 节点渲染宽度（在切线方向上的投影），可近似为 `node.width`。
- 计算父节点角度区间时，保证总弧长 ≤ `半径 × 总角度`（圆弧长度公式）。

简化版可以直接加大 `layerSpacing` 并限制子节点数量，或对扇区角度做最小宽度约束。

## 三、水平树状布局（向右展开的逻辑图）

根在左侧，子节点在右侧逐层垂直排列。适合组织结构图、向右展开的思维导图。

### 3.1 简单递归布局（节点尺寸固定）

思路：  
1. 后序遍历，计算每个子树的**总高度**（所有子节点高度 + 间距累加）。  
2. 对每个节点，将子节点在垂直方向上依次排布，整体居中于父节点。

```javascript
const HORIZONTAL_GAP = 80;   // 父子水平间距
const VERTICAL_GAP = 20;     // 兄弟垂直间距

function calcSubtreeHeight(node) {
  if (node.children.length === 0) {
    node.subtreeHeight = node.height;
    return node.subtreeHeight;
  }
  let totalHeight = 0;
  node.children.forEach(child => {
    totalHeight += calcSubtreeHeight(child);
  });
  totalHeight += VERTICAL_GAP * (node.children.length - 1);
  node.subtreeHeight = totalHeight;
  return totalHeight;
}

function horizontalLayout(node, x) {
  node.x = x;
  if (node.children.length === 0) {
    node.y = 0; // 相对父节点的偏移，后面会调整
    return;
  }

  // 子节点水平位置
  const childX = x + node.width + HORIZONTAL_GAP;

  // 计算子节点总高度
  let totalChildHeight = 0;
  node.children.forEach(child => {
    totalChildHeight += child.subtreeHeight;
  });
  totalChildHeight += VERTICAL_GAP * (node.children.length - 1);

  // 以父节点垂直中心为基准，从上往下排列子节点
  let startY = -totalChildHeight / 2;
  for (let child of node.children) {
    horizontalLayout(child, childX);
    // 将子节点子树整体移动到 startY + 子树高度一半的位置
    const offsetY = startY + child.subtreeHeight / 2;
    shiftSubtreeY(child, offsetY - child.y); // 修正绝对坐标
    startY += child.subtreeHeight + VERTICAL_GAP;
  }
}

function shiftSubtreeY(node, delta) {
  node.y += delta;
  node.children.forEach(child => shiftSubtreeY(child, delta));
}

// 根节点起始
calcSubtreeHeight(root);
horizontalLayout(root, 0);
// 根节点居中在 y=0
```

### 3.2 改进：优雅的树布局算法（Reingold-Tilford）

上述简单算法可能导致“兄弟子树之间挤在一起”。专业做法是使用 **Reingold-Tilford 算法**（或叫 Walker 算法），能产生紧凑且居中对齐的树布局。你可以在 GitHub 搜索 `tidy tree layout` 实现，或直接参考 D3.js 的 `d3.tree()` 源代码。

## 四、连线绘制

布局只是确定了节点坐标，连线同样影响视觉效果。推荐使用 **二次/三次贝塞尔曲线**：

- **辐射状**：从父节点边缘沿径向出发，弯曲连接到子节点边缘。可通过极坐标插值生成贝塞尔控制点。
- **水平树状**：使用 `S` 形曲线，比如从父节点右侧 → 控制点1 (父.x+间距/2, 父.y) → 控制点2 (子.x-间距/2, 子.y) → 子节点左侧。

伪代码示例（水平树状二次贝塞尔）：

```javascript
function getLinePath(parent, child) {
  const midX = (parent.x + parent.width/2 + child.x - child.width/2) / 2;
  return `M ${parent.x + parent.width/2} ${parent.y}
          Q ${midX} ${parent.y} ${midX} ${child.y}
          T ${child.x - child.width/2} ${child.y}`;
}
```

### **第一章：思维导图的本质与布局目标**

在着手编写任何布局代码之前，我们必须深刻理解思维导图的内在结构及其对可视化布局的根本要求。这不仅关乎技术选型，更决定了最终产品的用户体验。

#### **1.1 思维导图的根基：树形数据结构**

从数据结构的角度看，一个思维导图本质上是一个**树（Tree）** [[1]][[2]]。这个结构由以下几个核心元素构成：

*   **根节点（Root Node）：** 思维导图的中心主题，是所有分支的起点，也是树结构中唯一没有父节点的节点。
*   **节点（Node）：** 代表一个概念、想法或任务。每个节点可以包含文本、图片、链接等信息。
*   **父节点（Parent Node）与子节点（Child Node）：** 节点之间的层级关系。除根节点外，每个节点都有且仅有一个父节点。一个节点可以拥有零个或多个子节点。
*   **分支（Branch）与边（Edge）：** 连接父节点和子节点的线段，直观地展示了它们之间的从属关系。
*   **层级（Level）或深度（Depth）：** 节点在树中的位置。根节点位于第0层，其子节点位于第1层，以此类推。

理解这一核心结构至关重要，因为它意味着所有针对树形结构的可视化算法，原则上都可以应用于思维导图的布局 [[3]][[4]]。无论是简单的递归遍历还是复杂的高级算法，都是在对这个树形数据进行操作。

#### **1.2 自动布局的核心价值：解放创造力**

手动拖拽和对齐每一个节点对于用户而言是一项繁琐且低效的工作。一个优秀的思维导图软件，其核心竞争力之一便是提供强大而智能的**自动布局（Automatic Layout）**功能 [[5]][[6]]。其价值主要体现在：

1.  **聚焦内容创作：** 用户可以将全部精力投入到思考和梳理逻辑上，而无需分心于排版的美观性问题 [[7]]。软件应该“即时”地为用户生成一个清晰、优美的布局。
2.  **保证可读性与美观性：** 自动布局算法能够遵循既定的美学规则，确保即使在节点数量庞大、结构复杂的情况下，整个导图依然保持高度的可读性和视觉上的愉悦感。
3.  **动态适应性：** 当用户增加、删除、移动或编辑节点时，布局算法能够迅速响应，实时重新计算并调整相关节点的位置，维持整体结构的和谐与稳定。

因此，布局算法不仅仅是技术实现，更是产品设计理念的体现，直接影响着软件的可用性和用户的创作效率。

#### **1.3 布局的美学标准与设计目标**

一个成功的布局算法，其目标是生成“整洁”（Tidy）的绘图。这个看似主观的词汇，在图可视化领域被分解为一系列可量化的美学标准。为思维导图设计布局算法时，我们需要追求以下目标 [[8]]：

*   **节点不重叠（No Node Overlap）：** 这是最基本、最硬性的要求。任何两个节点的边界框（Bounding Box）都不能有任何交叉，以保证所有内容清晰可见。
*   **清晰的层级关系（Clear Hierarchy）：** 属于同一层级的节点应该在视觉上对齐。例如，在标准的从左到右布局中，同一深度的节点应该具有相同的X坐标。这有助于用户快速理解信息的组织结构。
*   **父节点中心化（Parent Centering）：** 父节点的位置应该尽可能地在其所有直接子节点的几何中心。例如，在一个垂直布局中，父节点的X坐标应约等于其子节点X坐标范围的中点 [[9]]。这能创造出一种视觉上的平衡感。
*   **空间紧凑性与利用率（Compactness and Space Utilization）：** 布局应该在保证清晰度的前提下，尽可能地节省空间。过于稀疏的布局会浪费屏幕资源，而过于紧凑则可能导致拥挤和混乱。算法需要在两者之间找到最佳平衡。
*   **连接线清晰（Edge Clarity）：**
    *   **避免边穿过节点：** 连接父子节点的边不应穿过任何不相关的节点。
    *   **最小化边交叉：** 虽然在复杂的图中完全避免边交叉很困难，但一个好的算法应力求减少边的交叉数量，以降低视觉混乱。
*   **结构对称性与平衡感（Symmetry and Balance）：** 如果一个节点的子树结构本身是对称的，那么布局结果也应该反映出这种对称性。整个思维导图应给人一种稳定、平衡的视觉感受。
*   **顺序保留（Order Preservation）：** 子节点的排列顺序应与其在数据结构中的顺序保持一致。如果用户将“第一步”放在“第二步”之前，布局也必须遵循这个顺序。

这些标准构成了我们评判和设计布局算法的准则。在后续章节中，我们将看到不同的算法是如何在这些标准之间进行权衡和优化的。

### **第二章：主流思维导图布局算法分类与解析**

思维导图软件通常支持多种布局模式，以适应不同的应用场景和用户偏好 [[10]][[11]]。这些布局背后是不同类别的算法。本章将对几种主流的布局算法进行分类和解析。

#### **2.1 标准树形布局 (Standard Tree Layout)**

这是最经典、最常用的布局方式，直接反映了思维导图的层级结构。它本身又包含多种变体。

*   **描述：** 这种布局将根节点放置在一个固定位置（如画布左侧或顶部），然后其子节点在另一侧（右侧或下方）依次展开。每一层级的节点在布局方向上（水平或垂直）具有相同的坐标。
*   **变体：**
    1.  **标准结构图（向右）：** 根节点在左，所有分支向右延伸。这是最常见的形式。
    2.  **树状图（向下）：** 根节点在上，所有分支向下延伸，类似于组织结构图。
    3.  **逻辑结构图（左右分布）：** 根节点居中，其子节点被平均分配到左右两侧，形成一个对称的、向两边伸展的结构 [[12]]。这种布局在视觉上非常平衡，空间利用率也较高。
    4.  **缩进布局（Indented Layout）：** 类似于文件资源管理器的列表视图。所有节点在垂直方向上排列，通过水平方向的缩进来表示层级关系 [[13]]。这种布局非常紧凑，适合展示长列表式的层级信息。
*   **实现原理：** 其核心是**分层布局（Layered Layout）**思想 [[14]]。通过树的深度优先遍历（DFS），可以确定每个节点的层级（depth）。节点的坐标一个维度由层级决定（例如，`x = depth * horizontal_spacing`），另一个维度则需要在同层级的兄弟节点中进行分配，同时要避免不同分支的子树发生重叠。这部分的具体计算是第三章和第四章的重点。
*   **优点：** 结构清晰，逻辑性强，完美地展现了信息的层级和顺序。
*   **缺点：** 当树的宽度或深度非常大时，可能会导致布局在某一方向上过度延伸，占用大量空间。

#### **2.2 径向布局 (Radial Layout / Sunburst Layout)**

径向布局提供了一种以中心为焦点、向外发散的视觉呈现方式，非常符合“中心主题-发散思维”的隐喻。

*   **描述：** 根节点位于画布的中心。第一层子节点分布在以根节点为圆心的第一个同心圆上；第二层子节点分布在第二个同心圆上，以此类推 [[15]][[16]][[17]]。每个节点的位置由其所在的层级（决定半径）和在同层级中的顺序（决定角度）共同决定。
*   **实现原理：**
    1.  **确定极坐标：** 首先，需要通过某种方式为每个节点分配一个极坐标 `(radius, angle)`。
        *   `radius` 通常与节点的层级成正比：`radius = node.depth * radius_per_level`。
        *   `angle` 的分配则较为复杂。一种简单的方法是，将360度平均分给第一层的子节点。对于更深层的节点，可以将父节点所占有的角度扇区，再按比例分配给它的子节点们。
    2.  **转换为笛卡尔坐标：** 得到极坐标后，使用三角函数将其转换为屏幕上用于渲染的笛卡尔坐标 `(x, y)`：
        *   `x = center_x + radius * cos(angle)`
        *   `y = center_y + radius * sin(angle)`
*   **优点：** 视觉冲击力强，能够有效突出中心主题的核心地位。在展示广度优先的层次结构时效果很好 [[18]]。
*   **缺点：**
    *   **空间效率低：** 随着层级增加，外层圆的周长迅速增大，导致节点间距变大，空间利用率下降。
    *   **方向性差：** 节点的文字方向需要特殊处理，否则可能出现倒置或倾斜，影响阅读。
    *   **易于重叠：** 如果不对角度进行精细计算，深层子树的节点很容易与相邻分支的子树重叠。

#### **2.3 力导向布局 (Force-Directed Layout)**

力导向布局是一种模拟物理世界的有机布局算法，它不依赖于固定的层级结构，而是通过模拟粒子间的相互作用力来找到一个“自然”平衡的布局。

*   **描述：** 该算法将整个图看作一个物理系统 [[19]][[20]]。
    *   **节点（Node）：** 被视为互相排斥的粒子（如带同种电荷的粒子），它们之间存在**斥力（Repulsion Force）**。这使得节点之间会尽量分开，避免重叠。
    *   **边（Edge）：** 被视为连接节点的弹簧，它们之间存在**引力（Attraction Force）**。这使得有连接关系的节点会互相靠近。
*   **实现原理：** 这是一个迭代过程 [[21]][[22]]。
    1.  **初始化：** 随机地将所有节点放置在画布上。
    2.  **迭代计算：** 在每一步迭代中：
        *   计算每对节点之间的斥力（通常与距离平方成反比）。
        *   计算每条边上两个节点之间的引力（通常与距离成正比）。
        *   将作用在每个节点上的所有力（斥力和引力）合成为一个合力。
        *   根据合力的方向和大小，微小地移动每个节点的位置。
    3.  **达到平衡：** 重复迭代过程，直到整个系统的总能量趋于稳定，即所有节点的移动幅度都非常小，此时布局达到一个平衡状态。
*   **优点：**
    *   **有机美感：** 能产生非常自然、平滑的布局，尤其适合展示网络结构和关系复杂的图。
    *   **结构自适应：** 能够自动发现图中的“社区”（Community）结构，将连接紧密的节点聚集在一起。
*   **缺点：**
    *   **不保证层级：** 无法像树形布局那样清晰地展示层级关系，父子节点的位置关系不固定。对于严格的思维导图来说，这通常是不可接受的。
    *   **结果不唯一：** 每次运行的结果可能略有不同，因为它依赖于初始位置和迭代过程。
    *   **计算量大：** 算法复杂度较高（经典实现为O(N²)或通过优化达到O(N log N)），在节点数量非常多时，计算时间会显著增加。

#### **2.4 其他专用布局**

除了上述三大类，思维导图软件还可能提供一些针对特定场景的布局 [[23]][[24]]。

*   **鱼骨图（Fishbone Diagram）：** 也称因果图，用于分析问题的根本原因。其布局形似鱼骨，主干代表问题，大刺代表主要原因类别，小刺代表具体原因。这是一种高度定制化的树形布局。
*   **时间线（Timeline）：** 将节点沿着一条水平或垂直的轴线排列，轴线代表时间。节点的顺序和位置由其时间属性决定。这是一种一维的约束布局。

**小结：** 对于通用的思维导图软件开发，**标准树形布局**及其变体（特别是逻辑结构图）是必须实现的核心功能。径向布局可以作为一种富有吸引力的可视化补充。力导向布局则更适用于“概念图”或需要探索复杂关系的场景，而非传统的层级思维导图。接下来的章节，我们将聚焦于如何实现最高频、最核心的标准树形布局。

### **第三章：核心实现原理：从递归思想到坐标计算**

本章将深入探讨实现标准树形布局的编程思想和具体计算步骤。我们将从一个相对基础的、直观的递归方法入手，这有助于理解布局算法的本质，并为理解后续的高级算法打下基础。

#### **3.1 树的遍历：布局算法的基石**

要为树中的每个节点确定位置，首先必须能够访问到每一个节点。树的遍历算法，特别是**深度优先搜索（Depth-First Search, DFS）**，是布局计算的天然选择 [[25]]。在布局场景中，我们通常需要对树进行两次遍历：

1.  **第一次遍历（自底向上，Post-order Traversal）：**
    *   **过程：** 先递归地处理完一个节点的所有子节点，最后再处理该节点本身。
    *   **目的：** 这次遍历的主要任务是计算每个子树的**相对**布局和尺寸信息。例如，确定一个子树需要占据多大的垂直空间，以及其内部节点的相对`y`坐标。在这个阶段，我们还不关心节点的绝对`x`坐标，因为绝对位置依赖于其父节点和兄弟节点的位置。

2.  **第二次遍历（自顶向下，Pre-order Traversal）：**
    *   **过程：** 先处理当前节点，然后递归地处理其所有子节点。
    *   **目的：** 这次遍历根据父节点的**绝对**位置，来计算子节点的最终绝对位置。在第一次遍历中计算出的相对位置信息将在这里被转换为最终的屏幕坐标。

这种“先上浮（计算尺寸），后下沉（分配位置）”的两遍（Two-Pass）策略是许多经典树布局算法的核心思想。

#### **3.2 递归方法的应用：分而治之**

树的递归定义（一个节点和它的子树森林）使得递归函数成为解决树问题的优美范式 [[26]][[27]][[28]]。我们可以定义一个核心的布局函数，如 `calculateLayout(node)`，它的职责是计算以 `node` 为根的子树的布局。

一个简化的递归布局伪代码可能如下所示 (参照 [[29]] 和 [[30]] 的逻辑)：

```
// 全局或传入的参数
const nodeWidth = 100;
const nodeHeight = 50;
const horizontalSpacing = 80;
const verticalSpacing = 20;

function layout(rootNode) {
    // 第一次遍历：计算初步的Y坐标和子树高度
    firstPass(rootNode, 0);

    // 第二次遍历：根据父节点位置计算最终的X, Y坐标
    // 假设根节点位于 (0, 0)
    rootNode.x = 0;
    rootNode.y = 0;
    secondPass(rootNode);
}

// 第一次遍历：后序遍历计算y偏移和高度
function firstPass(node, level) {
    node.level = level;
    node.y = 0; // 这是一个相对于父节点的初始y偏移

    let childrenTotalHeight = 0;
    if (node.children.length > 0) {
        // 递归处理所有子节点
        for (const child of node.children) {
            firstPass(child, level + 1);
        }

        // 计算子节点所占的总高度（包括间距）
        childrenTotalHeight = node.children.reduce((sum, child) => sum + child.subtreeHeight, 0) + 
                              (node.children.length - 1) * verticalSpacing;
        
        // 分配子节点的相对y坐标
        let currentY = -childrenTotalHeight / 2;
        for (const child of node.children) {
            child.y = currentY + child.subtreeHeight / 2;
            currentY += child.subtreeHeight + verticalSpacing;
        }
    }
    
    // 节点的子树高度 = 自身高度 或 子节点总高度，取较大者
    node.subtreeHeight = Math.max(nodeHeight, childrenTotalHeight);
}


// 第二次遍历：前序遍历计算最终坐标
function secondPass(node) {
    // 计算子节点的X坐标
    const childX = node.x + nodeWidth + horizontalSpacing;
    
    // 遍历子节点，设置它们的最终坐标
    for (const child of node.children) {
        child.x = childX;
        // 子节点的最终Y = 父节点的Y + 子节点相对父节点的Y偏移
        child.y = node.y + child.y; 

        // 递归对子树进行第二次遍历
        secondPass(child);
    }
}
```


#### **3.3 经典树形布局的坐标计算详解**

让我们将上述伪代码的逻辑分解为更详细的步骤。假设我们正在实现一个从左到右的布局。

**输入数据：**
*   每个节点对象，包含 `children` 数组。
*   每个节点的内容，用于计算其尺寸 `width` 和 `height` [[31]]。为简化，本节假设所有节点尺寸固定。

**输出数据：**
*   为每个节点对象赋予 `x` 和 `y` 坐标属性。

**步骤 1：定义常量与节点尺寸**
在计算开始前，必须明确布局的视觉参数 [[32]]。
*   `nodeWidth`, `nodeHeight`: 节点的宽度和高度。
*   `horizontalSpacing`: 节点在水平方向上的间距，即父子节点之间的距离。
*   `verticalSpacing`: 兄弟节点在垂直方向上的间距。

**步骤 2：第一次遍历 (Post-order) - 计算Y偏移和子树高度**

这次遍历的核心目标是确定每个节点相对于其父节点的**垂直偏移量（Y-offset）**，并计算出每个节点作为根的子树所占据的总垂直高度。

1.  **从叶节点开始：** 对于一个没有子节点的叶节点，它的 `subtreeHeight` 就是它自己的 `nodeHeight`。
2.  **处理内部节点：** 当一个节点的所有子节点都已处理完毕后，我们可以计算该节点的布局信息：
    a. **计算子树群总高度：** 将所有子节点的 `subtreeHeight` 相加，并加上它们之间的 `verticalSpacing`。例如，如果有3个子节点，就需要加 `2 * verticalSpacing` [[33]][[34]]。
       `totalChildrenHeight = sum(child.subtreeHeight for child in node.children) + (num_children - 1) * verticalSpacing`
    b. **确定父节点的美学位置：** 为了让父节点垂直居中于其子节点群，我们需要将子节点群的几何中心对准父节点的Y坐标 [[35]]。
    c. **分配子节点的相对Y坐标：** 计算第一个子节点的起始Y坐标。这个起始点应该是 `-(totalChildrenHeight / 2)`，这样整个子节点群就会以父节点的Y=0为中心对称分布。然后，从上到下依次放置每个子节点 [[36]]。
       *   第一个子节点的中心 `y_offset = -(totalChildrenHeight / 2) + child1.subtreeHeight / 2`
       *   第二个子节点的中心 `y_offset = (第一个子节点的y_offset) + child1.subtreeHeight / 2 + verticalSpacing + child2.subtreeHeight / 2`
       *   ...以此类推。
    d. **计算当前节点的子树高度：** 当前节点 `node` 的 `subtreeHeight` 是它自身的高度 `nodeHeight` 和其子树群总高度 `totalChildrenHeight` 中的较大值。`node.subtreeHeight = max(nodeHeight, totalChildrenHeight)`。这确保了即使父节点本身很高，也不会与子树群的边界框发生冲突。

经过这次遍历，每个节点都有了一个 `subtreeHeight` 属性，并且每个非根节点都有了一个相对于其父节点的 `y` 偏移量。

**步骤 3：第二次遍历 (Pre-order) - 计算最终绝对坐标**

这次遍历将相对位置转换为画布上的绝对坐标。

1.  **从根节点开始：** 赋予根节点一个初始位置，例如 `(0, 0)`。
2.  **处理子节点：** 对于当前正在访问的节点 `node`（已知其绝对坐标 `node.x`, `node.y`）：
    a. **计算子节点的X坐标：** 它的所有子节点的X坐标都是相同的，由父节点的X坐标、宽度和间距决定：
       `child.x = node.x + nodeWidth + horizontalSpacing`
    b. **计算子节点的Y坐标：** 子节点的绝对Y坐标等于父节点的绝对Y坐标加上在第一次遍历中计算出的相对Y偏移量：
       `child.y = node.y + node.children[i].y_offset`
    c. **递归调用：** 以每个子节点为参数，递归地执行此过程，直到所有节点都被赋予了最终的绝对坐标。

#### **3.4 挑战：子树重叠问题**

上述的简单递归方法有一个致命的缺陷：它没有考虑**不同分支的子树之间可能发生的重叠**。

**问题场景：** 假设节点A有两个子节点B和C。B有一个非常宽的子树，而C有一个很窄的子树。按照上述算法，B和C被垂直放置，它们的子树向右延伸。如果B的子树的右侧部分与C的子树的左侧部分在垂直上没有完全错开，它们就可能会重叠。

**解决方案雏形：**
为了解决这个问题，我们需要在放置一个子树时，考虑它之前所有兄弟子树所占据的空间。这需要在第一次遍历中引入更复杂的逻辑：
*   在放置节点 `C` 的子树时，不能简单地认为它的起始 `y` 坐标就是节点 `B` 的子树下方。
*   我们需要追踪整个兄弟节点群的**轮廓（Contour）**。当放置一个新的子树时，需要计算它的左轮廓与已放置的兄弟子树群的右轮廓之间的距离。如果距离小于要求的间距，就需要将新的子树整体向下移动，直到满足间距为止。

这种对轮廓的追踪和子树的移动，正是更高级算法如 Reingold-Tilford 算法和 Walker 算法所要解决的核心问题。简单的实现方式可能导致算法复杂度退化到 O(n²)，因为每次移动一个子树可能需要更新其所有后代节点的位置。这自然而然地引出了我们对更高效算法的探讨。

### **第四章：高级算法深度剖析：从Walker到Buchheim**

简单的递归布局算法在处理复杂或不平衡的树时，要么会产生重叠，要么为了避免重叠而导致布局过于稀疏，浪费空间。为了生成既紧凑又美观的“整洁”（Tidy）树状图，研究者们开发了一系列更精密的算法。本章将深入探讨这一领域的里程碑式算法，尤其是目前被广泛认为是最佳实践的 Buchheim 算法。

#### **4.1 算法的演进：对“整洁”与“效率”的追求**

*   **Knuth 算法：** 高德纳（Donald Knuth）提出的早期算法是基础，但它倾向于将节点放置在离散的整数坐标上，可能无法产生最紧凑的布局。
*   **Reingold-Tilford (RT) 算法：** 1981年，Reingold和Tilford提出了一种革命性的算法，能够在O(n)时间内为二叉树生成“整洁”的布局。其核心思想是：
    1.  递归地布局左右子树。
    2.  将右子树尽可能地向左移动，直到它碰到左子树。
    3.  将父节点放置在子节点上方居中的位置。
    这个算法奠定了现代树布局算法的基础，但其原始形式仅适用于二叉树。

#### **4.2 Walker 算法：从二叉树到多叉树**

1990年，John Q. Walker II 将 RT 算法的思想推广到了多叉树（即每个节点可以有任意数量的子节点），这对于思维导图等通用树结构至关重要 [[37]][[38]]。

*   **核心思想：** Walker 算法同样采用后序遍历（第一次遍历）。在处理一个节点的子节点时，它会依次放置每个子树。每当放置一个新的子树时，它会检查这个子树与它左侧所有已放置的兄弟子树之间是否存在冲突。如果存在，就将新子树向右移动，直到冲突消除。
*   **面临的挑战：** Walker 算法的关键和难点在于如何高效地检查和解决冲突。为了做到这一点，算法需要维护每个子树的**轮廓（Contour）**。当放置新子树 `C_i` 时，需要将其左轮廓与左边所有兄弟子树 `C_1, ..., C_{i-1}` 组成的聚合体的右轮廓进行比较。在最坏的情况下，例如一个深度很深、形如“梳子”的树结构，每次比较轮廓可能需要遍历树的深度，导致整个算法的时间复杂度退化为 **Ω(n²)** [[39]][[40]]。对于大型思维导图，平方级的复杂度是不可接受的。

#### **4.3 Buchheim 线性时间算法：现代思维导图布局的基石**

2002年，Christoph Buchheim, Michael Jünger, 和 Sebastian Leipert 发表了一篇论文，对 Walker 算法进行了关键性的改进，成功地将时间复杂度降低到了**线性时间 O(n)** [[41]][[42]][[43]]。这个改进后的算法，有时仍被泛称为 Walker 算法，是目前绝大多数高性能图表库（如 D3.js 的树布局）实现的基础 [[44]][[45]][[46]]。

Buchheim 算法的精髓在于引入了两个巧妙的数据结构来避免 Walker 算法中的重复计算：**`mod`（modifier）** 和 **`thread`**。

*   **核心优化1：`mod` - 惰性位移**
    *   **问题：** 在 Walker 算法中，当一个子树需要向右移动时，需要递归地更新该子树中所有节点的x坐标，这是一个成本高昂的操作。
    *   **Buchheim的方案：** 不立即更新所有后代节点的坐标。而是在每个节点上增加一个 `mod` 属性。`mod` 代表了该子树相对于其父节点坐标系的**累积偏移量**。
    *   **工作方式：** 当子树 `T` 需要向右移动 `d` 距离时，只需更新 `T` 根节点的 `mod` 值：`T.root.mod += d`。该子树中任意一个节点 `v` 的初步 `x` 坐标（在第一次遍历中计算）加上其所有祖先（直到 `T` 的根）的 `mod` 值之和，才是它相对于 `T` 根的真正 `x` 坐标。最终的绝对坐标将在第二次（前序）遍历中，通过累加从根节点到当前节点的路径上所有 `mod` 值来一次性计算出来。这样，一次子树移动操作的复杂度从 O(subtree_size) 降到了 O(1)。

*   **核心优化2：`thread` - 高效轮廓访问**
    *   **问题：** Walker 算法中，为了找到两个相邻子树的冲突点，需要在两个子树的轮廓上同步向下追溯，这在深层树上很慢。
    *   **Buchheim的方案：** 使用**线程（thread）**指针。当一个节点的子节点布局完成后，如果它内部的轮廓（例如，最右侧分支的最左侧节点，或最左侧分支的最右侧节点）在下一层没有真正的节点，就用一个 `thread` 指针指向其下一个轮廓上的兄弟节点。
    *   **工作方式：** 这些 `thread` 指针就像快捷方式，将一个子树的左右轮廓链接成一个链表。这样，在比较相邻子树的轮廓时，就可以通过这些 `thread` 指针在 O(1) 的时间内跳到下一层的轮廓节点，而无需进行深度遍历。这彻底消除了导致平方级复杂度的性能瓶颈。

#### **4.4 Buchheim-Walker 算法流程概述**

结合了 `mod` 和 `thread` 优化的完整算法流程如下：

**第一次遍历 (I-Pass, Post-order DFS)** - 函数 `layout(node)`:

1.  **基本情况：** 如果 `node` 是叶节点，直接返回。
2.  **递归调用：** 对 `node` 的所有子节点 `child`，递归调用 `layout(child)`。
3.  **子树初始放置：** 子节点完成布局后，它们被临时放置，使得每个子树的根节点 `x` 坐标都为0。它们之间的 `y` 坐标根据深度确定。
4.  **兄弟间位移：** 从第二个子节点开始，依次处理每个子节点 `v`：
    *   获取 `v` 和它左边的兄弟节点群 `w`。
    *   使用 `thread` 指针高效地沿着 `v` 的左轮廓和 `w` 的右轮廓向下检查，计算它们之间的最小距离，并确定需要将 `v` 向右移动多少（`shift`）。
    *   更新 `v` 的 `mod` 值：`v.mod += shift`。同时，更新 `v` 的初步 `x` 坐标。
    *   将 `v` 的轮廓信息合并到 `w` 的轮廓中，为下一个兄弟节点的比较做准备。
5.  **父节点居中：** 所有子节点的位置确定后，计算子节点群的中心点，并将父节点 `node` 的初步 `x` 坐标设置在该中心点之上。
6.  **设置线程：** 建立必要的 `thread` 指针以加速上层节点的轮廓计算。

**第二次遍历 (II-Pass, Pre-order DFS)** - 函数 `calculateFinalCoordinates(node, mod_sum)`:

1.  **计算最终坐标：**
    *   `node.x = node.prelim_x + mod_sum`
    *   `node.y = node.depth * layer_height`
2.  **递归调用：** 对 `node` 的所有子节点 `child`，递归调用 `calculateFinalCoordinates(child, mod_sum + node.mod)`。

虽然从零开始完整实现 Buchheim 算法细节繁多，但已有许多优秀的开源库可供参考和使用：

*   **JavaScript (D3.js):** D3.js 的 `d3-hierarchy` 模块中的 `tree()` 布局就是基于 Buchheim 算法的实现。其源码是学习该算法在Web前端应用的绝佳范例。
*   **Java (TreeLayout):** 搜索结果中提到的 `TreeLayout` 是一个专门用于此目的的Java库，它明确指出是基于Buchheim等人对Walker算法的增强实现的 [[47]]。
*   **Android (Android TreeView):** 这个库为安卓平台提供了树视图，并且支持包括Walker算法（含Buchheim改进）在内的多种布局算法 [[48]]。
*   **Flutter (GraphView):** Flutter的 `GraphView` 库也明确实现了基于Buchheim改进的Walker算法，用于高效地绘制树形图 [[49]]。

对于想要自己实现思维导图软件的开发者，强烈建议深入研究上述一个或多个库的源代码。理解它们如何管理节点数据结构（特别是 `mod`, `thread`, `prelim_x` 等字段）以及如何组织两次遍历的逻辑，是成功实现该算法的关键。

### **第五章：扩展与定制：实现多样化的思维导图布局**

掌握了核心的树形布局算法之后，我们就可以在此基础上进行扩展，以支持更丰富的布局模式和功能。

#### **5.1 实现逻辑布局（左右分布）**

逻辑布局是思维导图中最受欢迎的布局之一，它将根节点置于中心，子节点均匀分布于左右两侧。

**实现思路：**

1.  **子节点分组：** 将根节点的直接子节点分为两组：左侧组和右侧组。可以简单地按奇偶索引分配，或者允许用户手动指定。
2.  **独立布局：**
    *   对**右侧组**的节点，将它们构造成一棵新的树（或子树），然后运行标准的从左到右的Buchheim-Walker布局算法。
    *   对**左侧组**的节点，同样构造成一棵树。然后，运行一个**修改版**的布局算法。最简单的方式是，正常运行从左到右的算法，然后在最后一步将所有计算出的 `x` 坐标取反 (`x = -x`)。这样就得到了一个从右到左的镜像布局。
3.  **合并与定位：**
    *   将根节点放置在 `(0, y_root)`。
    *   将右侧布局的子树群的左边界与根节点的右边界对齐（加上间距）。
    *   将左侧布局的子树群的右边界与根节点的左边界对齐（加上间距）。
    *   可能需要对左右子树群进行整体的垂直微调，以确保它们的视觉中心对齐。

#### **5.2 实现径向布局**

如第二章所述，径向布局将节点放置在同心圆上。一种高质量的径向布局算法可以复用树形布局的结果。

**实现思路（基于先前的树形布局）：**

1.  **第一次布局：** 首先，对整个树运行一次标准的**自顶向下**的树形布局算法（注意，不是左右布局，而是类似组织结构图的向下布局）。这次布局的目的不是为了直接使用 `x`, `y` 坐标，而是为了获得两个关键信息：
    *   每个节点的**层级（depth）**。
    *   在同一层级中，每个节点的**水平顺序**（由`x`坐标决定）。
2.  **转换为极坐标：**
    *   **半径（Radius）：** `radius = node.depth * level_spacing`。`level_spacing` 是每个同心圆之间的距离。
    *   **角度（Angle）：** 角度的分配是关键。不能简单地将360度均分给所有叶子节点，这样会导致中间层节点拥挤。一种更好的方法是，首先为所有**叶子节点**在 `[0, 2π]` 范围内分配一个角度（可以根据它们在步骤1中得到的`x`坐标按比例分配）。然后，一个**非叶子节点**的角度，被定义为它所有子节点角度的**平均值**。这种自底向上的角度计算方法（类似于第一次遍历）可以确保父节点大致位于其子树扇区的中心。
3.  **转换为笛卡尔坐标：**
    *   `node.x = center_x + radius * cos(angle)`
    *   `node.y = center_y + radius * sin(angle)`

这种方法比直接在极坐标系中解决重叠问题要简单得多，并且能产生非常美观的径向布局。

#### **5.3 应对动态节点尺寸**

在实际应用中，节点的大小通常由其内容（如文字长度）决定，是动态变化的。这给布局算法带来了新的挑战。

**修改算法以适应动态尺寸：**

1.  **预计算尺寸：** 在进行任何布局计算之前，必须先遍历一次所有节点，根据其内容（文本、图片等）计算出每个节点的精确宽度 `node.width` 和高度 `node.height`。这通常需要调用UI框架的文本测量API。
2.  **调整坐标计算：**
    *   在Buchheim-Walker算法中，`y` 坐标的计算（`node.y = node.depth * ...`）需要被修改。不能再使用固定的层高，而应该让每一层的深度位置取决于该层及以上所有层的**最大节点高度**之和。
    *   在计算 `x` 坐标时，父子节点间的水平间距不再是 `node.width + horizontalSpacing`，而应该是 `parent.width/2 + child.width/2 + horizontalSpacing`，这样计算的是节点中心点之间的距离。
    *   在检测轮廓冲突时，比较的不再是节点的中心线，而是节点的**边界框（Bounding Box）**。算法需要维护子树轮廓的真实边界，而不仅仅是中心线的位置。

虽然这增加了实现的复杂性，但基本算法框架（两次遍历、`mod`、`thread`）依然适用。许多现代图表库，如 `@plait/layouts` [[50]] 和 `d3-flextree` [[51]]，都提供了对动态节点尺寸的支持。

### **结论与建议**

## 思维导图布局算法详解

思维导图的布局本质上是**树形图的绘制问题**，核心挑战是：如何让节点不重叠、连线美观、整体紧凑。

### 一、主流布局算法

#### 1. Reingold-Tilford 算法（最经典）

这是思维导图最常用的基础算法，分两遍遍历树：

**第一遍（后序遍历）：** 自底向上计算每个子树的初始位置
- 叶节点 x=0
- 非叶节点的 x = 第一个子节点与最后一个子节点的中点
- 兄弟节点之间保持最小间距

**第二遍（前序遍历）：** 自顶向下累加偏移量
- 将父节点的绝对位置传递给子节点

```
核心思路伪代码：
function layout(node, depth):
    if node is leaf:
        node.y = depth * levelHeight
        node.x = nextX
        nextX += nodeWidth + gap
    else:
        for child in node.children:
            layout(child, depth + 1)
        # 父节点居中于子节点
        node.x = (firstChild.x + lastChild.x) / 2
        node.y = depth * levelHeight
```

#### 2. Walker 改进算法

Reingold-Tilford 在子树宽度差异大时会有问题，Walker 算法增加了**子树间距修正**：

- 引入 `shift` 概念：当左右子树轮廓冲突时，计算最小平移量
- **轮廓（contour）**：左子树的右轮廓 vs 右子树的左轮廓，逐层比较找最小间距

```
function computeShift(leftTree, rightTree):
    leftContour = getRightContour(leftTree)
    rightContour = getLeftContour(rightTree)
    maxOverlap = 0
    for each level:
        overlap = leftContour[level] + gap - rightContour[level]
        maxOverlap = max(maxOverlap, overlap)
    return maxOverlap
```

#### 3. 思维导图专用：中心辐射布局

这是思维导图最典型的布局——根节点居中，分支向左右两侧展开：

```
核心逻辑：
1. 将根节点的子节点分为左右两组
2. 左侧组：从右到左排列，分支向左展开
3. 右侧组：从左到右排列，分支向右展开
4. 每侧内部用 Reingold-Tilford 或 Walker 算法布局
```

### 二、实战代码框架（JavaScript）

```javascript
class MindMapLayout {
  constructor(options = {}) {
    this.nodeWidth = options.nodeWidth || 120;
    this.nodeHeight = options.nodeHeight || 40;
    this.horizontalGap = options.horizontalGap || 60;  // 水平间距
    this.verticalGap = options.verticalGap || 16;      // 垂直间距
  }

  layout(root) {
    // 第一步：将子节点分为左右两组
    this.splitChildren(root);
    
    // 第二步：分别计算左右子树
    this.layoutSubtree(root.rightChildren, 'right');
    this.layoutSubtree(root.leftChildren, 'left');
    
    // 第三步：定位根节点
    root.x = 0;
    root.y = 0;
    
    return root;
  }

  splitChildren(root) {
    const mid = Math.ceil(root.children.length / 2);
    root.rightChildren = root.children.slice(0, mid);
    root.leftChildren = root.children.slice(mid);
  }

  layoutSubtree(children, direction) {
    // 计算每个子树的高度
    let yOffset = 0;
    for (const child of children) {
      const subtreeHeight = this.getSubtreeHeight(child);
      this.positionNode(child, direction, yOffset);
      yOffset += subtreeHeight + this.verticalGap;
    }
    // 整体居中
    const totalHeight = yOffset - this.verticalGap;
    const offset = -totalHeight / 2;
    this.shiftY(children, offset);
  }

  getSubtreeHeight(node) {
    if (!node.children || node.children.length === 0) {
      return this.nodeHeight;
    }
    let total = 0;
    for (const child of node.children) {
      total += this.getSubtreeHeight(child);
      total += this.verticalGap;
    }
    return total - this.verticalGap;
  }

  positionNode(node, direction, yOffset) {
    const sign = direction === 'right' ? 1 : -1;
    node.x = sign * this.horizontalGap;
    node.y = yOffset;
    
    if (node.children && node.children.length > 0) {
      let childYOffset = 0;
      const subtreeH = this.getSubtreeHeight(node);
      childYOffset = yOffset - subtreeH / 2;
      
      for (const child of node.children) {
        const childH = this.getSubtreeHeight(child);
        child.x = node.x + sign * this.horizontalGap;
        child.y = childYOffset + childH / 2;
        childYOffset += childH + this.verticalGap;
        this.positionNode(child, direction, child.y - childH / 2);
      }
    }
  }
}
```

### 三、连线绘制（贝塞尔曲线）

思维导图的连线通常用**三次贝塞尔曲线**，让分支看起来自然流畅：

```javascript
function drawConnection(ctx, parent, child) {
  const startX = parent.x + (child.x > parent.x ? parent.width / 2 : -parent.width / 2);
  const startY = parent.y;
  const endX = child.x + (child.x > parent.x ? -child.width / 2 : child.width / 2);
  const endY = child.y;
  
  // 控制点：水平方向偏移
  const cpOffset = Math.abs(endX - startX) * 0.5;
  const cp1x = startX + (child.x > parent.x ? cpOffset : -cpOffset);
  const cp1y = startY;
  const cp2x = endX + (child.x > parent.x ? -cpOffset : cpOffset);
  const cp2y = endY;
  
  ctx.beginPath();
  ctx.moveTo(startX, startY);
  ctx.bezierCurveTo(cp1x, cp1y, cp2x, cp2y, endX, endY);
  ctx.stroke();
}
```

## 分支布局怎么写
核心思路是：先给每个节点分配“子树占用宽度”，再决定它放在父节点的哪一侧或哪个扇区。 [en.wikipedia](https://en.wikipedia.org/wiki/Radial_tree)

常见流程是：
1. 先遍历整棵树，计算每个节点子树的“尺寸”，通常是叶子数、节点数，或者基于文本宽度的实际像素宽度。 [g6.antv.antgroup](https://g6.antv.antgroup.com/en/manual/layout/mindmap-layout)
2. 给同一层节点分配统一的层间距，保证同深度节点在同一圈或同一条水平线上。 [en.wikipedia](https://en.wikipedia.org/wiki/Radial_tree)
3. 对每个父节点，把子节点按顺序分配到左/右两边，或按角度扇区分配到不同区域，避免重叠。 [en.wikipedia](https://en.wikipedia.org/wiki/Radial_tree)
4. 再做一次碰撞检测和微调，把同层节点间距拉开，必要时整体平移子树。 [stackoverflow](https://stackoverflow.com/questions/8289518/algorithm-for-efficiently-drawing-trees)

## 一个简单算法框架
你可以把它理解成三步：

- **第一步：计算子树权重**。

- **第二步：分配空间**。

- **第三步：递归落位**。  
  在自己的切片里，孩子再继续切分自己的子空间。

如果是左右思维导图，一级节点可以用“奇偶分组”或“按权重平衡分组”放左右两边；如果是径向图，就把同一父节点的孩子放进一个角度区间内平均分配。 [en.wikipedia](https://en.wikipedia.org/wiki/Radial_tree)

### 3.1 数据结构

```ts
interface TreeNode {
  id: string;
  width: number;   // 节点渲染后的宽度
  height: number;  // 节点渲染后的高度
  children: TreeNode[];
  // 计算出的布局结果
  x?: number;
```

### 3.2 三轮遍历布局（简单版本）

这个简单版来自一篇实现「逻辑结构图/右向树」的实践：  
- 第一轮：确定每个节点的 **left(x)**（横坐标）
- 第二轮：自底向上算每个节点的「子树总高度」
- 第三轮：自上而下给子节点分配 **top(y)**（纵坐标），并在必要时微调兄弟子树，避免重叠

#### 3.2.1 第一轮：确定 x（横坐标）

规则：

- 根节点放在画布中心（或你想要的初始位置）
- 子节点一律放在父节点右侧，一定的水平间距

伪代码示意：

```ts
const MARGIN_X = 40; // 父子水平间距

function firstPass(node: TreeNode, isRoot: boolean, canvasWidth: number) {
  if (isRoot) {
    node.x = (canvasWidth - node.width) / 2; // 简单放在中间
  }

  for (const child of node.children) {
    child.x = node.x! + node.width + MARGIN_X;
    firstPass(child, false, canvasWidth);
  }
}
```

> 这一轮先不管 y，只确定 x。

#### 3.2.2 第二轮：计算每个节点的「子树总高度」

我们希望父节点竖直居中于整棵子树，所以要知道所有子节点+间距占多高。

```ts
const MARGIN_Y = 20; // 垂直间距

function calcChildrenAreaHeight(node: TreeNode): number {
  if (!node.children.length) {
    // 没有子节点，就不占额外高度
    (node as any).childrenAreaHeight = 0;
    return node.height;
  }

  let totalChildrenHeight = 0;
  for (const child of node.children) {
    const subTreeHeight = calcChildrenAreaHeight(child);
    totalChildrenHeight += subTreeHeight;
  }

  // 子节点之间 & 上下的间距：(len + 1) * MARGIN_Y
  const len = node.children.length;
  const childrenAreaHeight =
    totalChildrenHeight + (len + 1) * MARGIN_Y;

  (node as any).childrenAreaHeight = childrenAreaHeight;

  // 整棵子树高度 = max(自身高度, 子树高度 + 上下 padding)
  const subTreeTotalHeight = Math.max(
    node.height,
    childrenAreaHeight - 2 * MARGIN_Y
  );
  (node as any).subTreeTotalHeight = subTreeTotalHeight;

  return subTreeTotalHeight;
}
```

#### 3.2.3 第三轮：自上而下计算 y（纵坐标）

规则：  
- 对于有子节点的父节点：
  - 先把子树整体竖直居中在父节点的中线附近
  - 然后在子树区域里，从上到下依次放每个子节点

伪代码：

```ts
function layoutY(node: TreeNode) {
  if (!node.children.length) return;

  const areaHeight = (node as any).childrenAreaHeight as number;
  // 父节点中心
  const parentCenterY = node.y! + node.height / 2;
  // 子树整体区域顶部
  let currentY = parentCenterY - areaHeight / 2 + MARGIN_Y;

  for (const child of node.children) {
    child.y = currentY;
    layoutY(child);
    // 下一个子节点的起点 = 当前子树高度 + 间距
    currentY += (child as any).subTreeTotalHeight + MARGIN_Y;
  }
}
```

有时会出现「某个父节点子树高度比自己高很多，导致兄弟子树互相挤压」的情况，可以在这轮之后加一段「修正兄弟 y 的函数」，逐级向上调整，这一部分参考「updateBrothers / updateChildren」的思路即可。

到这里，你就有一个**可用的右向思维导图布局**了。

## 4. 第二步：上一个档次——Buchheim 紧凑树算法

上面的简单版：

- 兄弟子树之间的间距可能比较浪费空间
- 对树的左右对称性、美观性控制不精细
- 不支持“同一层节点不一定严格对齐”的紧凑布局

更专业的思维导图软件多采用 **Reingold–Tilford → Walker → Buchheim** 这一条线的改进算法：  
核心关键词：

- **两次遍历**：`first_walk`、`second_walk`
- **mod（修正量）**：一个节点相对其子树中心的水平偏移
- **轮廓（contour）**：子树左/右边界
- **apportion**：对比相邻子树的轮廓，计算冲突并用线程 + mod 分摊
- **execute_shifts / move_subtree**：把冲突的移动量线性分摊到多棵子树上

### 4.1 节点结构（精简版）

```ts
class DrawTree {
  name: string;
  x = -1;
  y: number;
  children: DrawTree[];
  parent: DrawTree | null;
  thread: DrawTree | null = null;
  mod = 0;
  ancestor: DrawTree;
  change = 0;
  shift = 0;
  _lmost_sibling: DrawTree | null = null;
  number: number; // 在兄弟中的索引，从 1 开始

  constructor(tree: any, parent: DrawTree | null = null, depth = 0, number = 1) {
    this.name = tree.name;
    this.y = depth;
    this.parent = parent;
    this.children = (tree.children || []).map(
      (child: any, i: number) => new DrawTree(child, this, depth + 1, i + 1)
    );
    this.ancestor = this;
    this.number = number;
  }

  right() { /* 右轮廓下一点 */ }
  left() { /* 左轮廓下一点 */ }
  left_brother() { /* 左兄弟 */ }
  get_lmost_sibling() { /* 最左兄弟 */ }
}
```

### 4.2 第一遍：`first_walk`

- 叶子：  
  - 有左兄弟：`x = leftBrother.x + distance`  
  - 无左兄弟：`x = 0`
- 非叶子：  
  - 先递归所有子节点  
  - 计算子节点的中点 `midpoint`  
  - 有左兄弟：  
    - `x = leftBrother.x + distance`  
    - `mod = x - midpoint`  
  - 无左兄弟：  
    - `x = midpoint`

并在处理每个子节点后调用 `apportion(child)` 来修正相邻子树冲突。

### 4.3 第二遍：`second_walk`

- 把所有祖先节点的 `mod` 累加到 `x` 上
- `y` 通常就是深度

```ts
function second_walk(v: DrawTree, m = 0, depth = 0) {
  v.x += m;
  v.y = depth;
  for (const child of v.children) {
    second_walk(child, m + v.mod, depth + 1);
  }
}
```

### 4.4 如何适配成「思维导图」

前面的 tidy-tree 算法本身不关心「横向还是纵向」。你可以：

- 把算法中的 `x` 当作「**横向距离**」（深度方向）
- 把算法中的 `y` 当作「**纵向位置**」

如果你更喜欢「纵向为层级，横向为同层顺序」，可以在最后 swap 一下坐标。

## 5. 第三步：做成真正的「思维导图」布局

上面的算法只是「一棵树」的布局，你还需要加上一些思维导图特有的东西：

### 5.1 根节点左右分支

常见思维导图：根在中间，分支一半在左，一半在右。

一种简单做法：

1. 把根节点的 children 拆成 `leftChildren` 和 `rightChildren` 两组
2. 对右边那组，正常用右向树布局（x 增大）
3. 对左边那组，**先当成右向树布局算出坐标**，然后：
   - `x = root.x - (child.x - root.x)`（左右镜像）或整体取反
4. 注意左右两边要各自做一棵 tidy tree，再在根处拼起来即可

### 5.2 节点宽高不等

Buchheim 原始算法假设节点宽高相等，现实里思维导图每个节点宽度差异巨大。常见做法：

- 在 tidy-tree 中只处理「**中心线**」的逻辑坐标
- 最终像素坐标时，把逻辑坐标乘以一个单元宽度，加上节点自己的宽度的一半：
  - `screenX = depth * H_GAP + sum(widths of all ancestors)`
  - `screenY = logicalY * V_GAP`  
- 也可以引入第三、四次遍历（`third_walk/fourth_walk`）来计算子树的 `minY/maxY` 并做纵向偏移，以避免子树之间纵向重叠。

### 5.3 交互友好的局部重布局

真实产品里，每次拖拽/插入节点都全量重跑布局，会卡。

可借鉴一些现成实践：

- 每次只从「变化节点」向上到根，重算那一条路径及其兄弟子树
- 通过记录每个子树的包围盒（bounding box），判断是否需要移动兄弟
- 为了性能，把布局计算放到 Web Worker，主线程只做动画过渡

## 7. 总结：你可以这样开始写

1. 先写一个**简单右向树布局三遍历**，把一棵有根树排成右向树（父在左、子在右且竖直居中）
2. 再把这套逻辑抽象成一个 `layout(tree)` 函数，输入树结构，输出每个节点 `(x, y)`
3. 之后需要更高级效果，再：
   - 引入 Buchheim/Reingold–Tilford 的 tidy-tree 算法（两遍遍历 + apportion）
   - 增加根节点左右分支支持
   - 处理节点宽高不等（宽度累加法 / third_walk & fourth_walk）
   - 做局部重布局 + 动画过渡
